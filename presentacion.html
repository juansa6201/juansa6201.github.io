Córdoba, 7-10-2025

Project Charter:
ProveedoresArg

Justificación: Solucionar la fragmentación y falta de confianza en la búsqueda de proveedores mayoristas para minoristas en Argentina.

Objetivo: Lanzar una plataforma web verificada sin intermediarios ni comisiones. Meta inicial: 100 proveedores activos.

Project Manager
Benjamín Rodríguez
Duración Estimada
2 Meses

Alcance, Supuestos y Restricciones

Entregables (In Scope)

  • Sitio web funcional con registro.
  • Panel de administración completo.
  • Buscador avanzado (filtros).
  • Lanzamiento y difusión.

Supuestos del Proyecto

  • • Todos los integrantes completan tareas en tiempo.
  • • Conexión estable y acceso a entornos garantizado.
  • • Proveedores de prueba colaboran con datos reales.

Exclusiones

  • • App móvil nativa
  • • Pasarela de pago integrada
  • • Logística o envíos automatizados

Restricciones

3 Semanas
4 Personas

Estructura de Desglose de Trabajo (WBS)

1. Planificación

Definición de objetivos SMART, identificación de stakeholders y elaboración del plan de trabajo y cronograma.

2. Frontend

Diseño de interfaz (Astro/React). Implementación de registro "multistep", buscador con Algolia y optimización SEO.

3. Backend

Configuración DB (Laravel MySQL). API de proveedores. Integración almacenamiento nube (Cloudflare R2).

4. Lanzamiento

QA y corrección de errores. Publicación en Vercel. Carga de datos reales y difusión inicial en redes.

Cronograma y Requisitos Técnicos

Fase 1: Requisitos

Revisión de flujos y campos (razón social, CUIT).

Fase 2: Base de Datos

Tablas en PostgreSQL: proveedores, rubros y claves.

Fase 3: APIs

Endpoints REST con Node/Express para alta y validación.

Fase 4: Testing

Pruebas unitarias y despliegue en entorno de staging.

Fase 5: Documentación

Manual de uso, despliegue y feedback final.
